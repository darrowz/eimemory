# OpenClaw 2026.7.1 Compatibility Audit

**Audit date:** 2026-07-18

**Official baseline:** [OpenClaw v2026.7.1 release notes](https://docs.openclaw.ai/releases/2026.7.1)

**Observed production host:** `OpenClaw 2026.7.1-2 (0790d9f)` on honxin.

## Compatibility Matrix

| Official contract | EIMemory action | Verification |
| --- | --- | --- |
| Native plugins use one `openclaw.plugin.json` and package-owned runtime entrypoint | Keep only `integrations/openclaw/eimemory-bridge/openclaw.plugin.json`; remove the redundant parent manifest | Manifest regression |
| Package compatibility can require a plugin API floor | Declare `openclaw.compat.pluginApi` and peer compatibility as `>=2026.7.1` | Package regression and runtime inspect |
| Typed hooks include `before_tool_call`, `after_tool_call`, and `message_sent` | Synchronize the manifest with all runtime registrations | Hook-list and Node registration regressions |
| Runtime `registerTool` registrations match `contracts.tools` | Register both `eimemory_bridge_status` and `memory_e2e_check` by default | Runtime inspection requires exact two-tool equality |
| Raw conversation hooks require explicit access | Atomically set `plugins.entries.eimemory-bridge.hooks.allowConversationAccess=true` | Config preservation/idempotency regressions |
| Prompt access and prompt injection are separate risks | Preserve `allowPromptInjection`; the L5 gate remains its only enabler | Config regression and L5 tests |
| Plugin changes require a Gateway restart and runtime inspection | Refresh registry, restart Gateway, and run `plugins inspect ... --runtime --json` | Installer order regression and production acceptance |
| Outbound receipt proves platform acceptance, not display/read | Rename successful delivery state to `platform_accepted` | Delivery state-machine regressions |

## Relevant 2026.7.1 Changes

The release adds GPT-5.6 compatibility and Gateway/delivery reliability fixes.
The EIMemory bridge does not depend on model-specific GPT-5.6 request fields,
so no adapter change is required. It does depend on typed hook and plugin
runtime contracts, which are pinned and verified by this release.

Feishu remains a supported production channel. The 2026.7.1 fixes for content
truncation, oversized provider error payloads, reconnect cleanup, and failure
banner leakage are host-runtime fixes; EIMemory validates the installed host
version and exercises the actual message path in production acceptance.
