# eimemory 1.7.7 Weakness Closure Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Turn the current weak spots into closed-loop gates: monitored timers, user-correction replay, evidence-gated research/news, structured candidates, fixed operational skills, graph lessons, and metric sufficiency.

**Architecture:** Add small glue modules on the existing governance line instead of a parallel autonomy system. Every new behavior writes evidence records, has a replay/gate shape, and is exposed through Runtime/CLI only where an operator or timer needs to call it.

**Tech Stack:** Python 3.11+, existing `RecordEnvelope`/`Runtime`/SQLite store, user systemd units, stdlib `urllib` for optional Feishu webhook delivery, pytest.
