# eimemory 1.8.3 Isolated Evaluator Harness Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Prevent long-running autonomous learning from treating generator self-claims as verified facts by adding a physically isolated evaluator packet, skeptical verdict, and independent stop judge gate.

**Architecture:** Generator output becomes an `evaluation_packet` with the generator claim quarantined from evaluator context. The default model split is `generator=gpt`, `evaluator=minimax`, and `stop_judge=minimax`; pass requires the generator and evaluator model IDs to differ. The evaluator sees only the goal, artifact, replay/verification evidence, and model identity separation; it defaults to fail until real execution evidence passes. A stop judge reads the evaluator verdict plus debt metrics and decides `stop`, `continue`, `quarantine`, or `require_human`.

**Tech Stack:** Python dataclasses/dicts, existing `RecordEnvelope` store, autonomous learning loop, pytest, user-systemd deployment.
