# EIMemory 1.6.0 — Harness Patch Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Land a "harness-patch core" that lets eimemory evolve its own governance via `失败轨迹 → 小补丁 → 双重回归门 → 版本化升级`. The current capability_ledger records scores, but proposal patches are not versioned assets, regression_watch is a single-source soft threshold, and there is no per-surface audit trail. This plan closes those gaps based on the design principles in `docs/superpowers/specs/2026-06-22-harness-patch-design-principles.md` and the Self-Harness paper (arXiv 2606.09498).

**Architecture:** A new `harness_patch` module owns three new abstractions: `HarnessSurface` (5 enums: `INSTRUCTION`, `VERIFICATION_GUIDANCE`, `TOOL_LOOP_GUARD`, `ARTIFACT_RECOVERY`, `RUNTIME_POLICY`), `ProposalCard` (mandatory dataclass: target surface, evidence, expected delta, target agent, risk tier, rollback plan, diff size), and `HarnessGate` (dual-regression wrapper that requires `held-in` AND `held-out` AND `delta≥0`). Existing modules import these and add hard enforcement at the boundary: `capability_ledger` rejects any `proposal_*` candidate without a card, `candidate_search` rejects oversized or non-diverse candidates, `regression_watch` requires both splits to clear, `promotion_manager` rejects L3+ candidates without a wire to `governance/safety/`.

**Tech Stack:** Python 3.11+, pytest, existing eimemory runtime/store/governance modules, `held_out_split.py` for the 70/30 split (already implemented at 70/30 with seed=42), `safety/` modules already at `governance/safety/` (kill_switch, circuit_breaker, spend_guard, audit_verifier, l3_queue).

**Reference papers:**
- Self-Harness (arXiv 2606.09498, 6/8/2026) — primary inspiration
- MRAgent (arXiv 2606.06036, 6/4/2026) — explicitly OUT of scope, deferred to 1.7.0

## 2. Scope

### 2.1 In-Scope（必须做完才能叫 1.6.0）

1. New `harness_patch` module with `HarnessSurface` enum + `ProposalCard` dataclass + `HarnessGate` evaluator
2. `capability_ledger` rejects any candidate promotion without a `ProposalCard` (when `HARNESS_PATCH_V2=1`)
3. `candidate_search` rejects oversized diffs (line count > 50 OR token count > 2000), requires ≥3 diverse candidates per surface, and enforces "one active candidate per surface"
4. `regression_watch` upgrades from single-source soft threshold to `held-in ∩ held-out ∩ delta≥0` dual-source hard gate (when `HARNESS_PATCH_V2=1`)
5. `promotion_manager` rejects L3+ candidates without a wire to `governance/safety/` (kill_switch + circuit_breaker + spend_guard must all be present)
6. New `eimemory patch` CLI: `propose / validate / promote / rollback / list` 5 subcommands
7. `tests/test_harness_patch.py` covering the full propose → validate → promote → rollback path with 50+ test cases

### 2.2 Non-Goals（明确不做）

- ❌ MRAgent evidence loop / shadow loop / cue-tag-content / active reconstruction → **1.7.0**
- ❌ Action sequence clustering in `capability_attribution` (would need full trace re-keying) → **1.6.1**
- ❌ Weakness mining infrastructure (would need a mining runner + dedup + report) → **1.7.0**
- ❌ Outcome integration with `outcome_replay.py` (depends on mining) → **1.7.0**
- ❌ User-facing recall improvements (memory.recall path stays untouched)
- ❌ Knowledge engine changes (claims, relations, views stay untouched)
- ❌ Raw evidence API changes
- ❌ Model weight changes / decoding config changes
- ❌ New LLM dependencies in production (proposal generation is just a dataclass, no LLM call)

### 2.3 Risk Tiers（沿用设计文档 §4）

| Tier | Auto-promote | Required evidence |
|---|---|---|
| L0 observation | ✅ | failed evidence + regression gate |
| L1 local software | ✅ | L0 + 100% replay + scope match |
| L2 business change | ⚠️ gated rollout | L1 + health + canary + rollback validation + audit |
| L3 real account / privacy | ❌ human | L2 + human audit + adversarial test |
| L4 physical / financial / medical / legal | ❌ multi-sig | L3 + multi-sig + business approval |

L3+ must wire `kill_switch` + `circuit_breaker` + `spend_guard` + `audit_verifier`. This plan enforces that wire in `promotion_manager`.

"""Harness Patch — versioned asset for governance evolution.

Implements design principles in
``docs/superpowers/specs/2026-06-22-harness-patch-design-principles.md``.

Every governance candidate promotion MUST carry a ``ProposalCard``. The card
is enforced at:

* ``capability_ledger.record_capability_score`` (rejects writes missing the card)
* ``candidate_search.generate_candidate_policies`` (rejects oversized / non-diverse groups)
* ``regression_watch.evaluate_harness_gate`` (requires held-in ∩ held-out ∩ delta≥0)
* ``promotion_manager.promote_candidate`` (rejects L3+ without safety wire)
"""
from __future__ import annotations

import os
from dataclasses import dataclass, field
from enum import Enum
from typing import Any


# Backwards-compat: opt-in via env var so 1.5.x callers are unaffected.
HARNESS_PATCH_V2 = os.environ.get("HARNESS_PATCH_V2") == "1"


class HarnessSurface(str, Enum):
    """The five editable surfaces of an agent harness."""

    INSTRUCTION = "INSTRUCTION"
    VERIFICATION_GUIDANCE = "VERIFICATION_GUIDANCE"
    TOOL_LOOP_GUARD = "TOOL_LOOP_GUARD"
    ARTIFACT_RECOVERY = "ARTIFACT_RECOVERY"
    RUNTIME_POLICY = "RUNTIME_POLICY"


@dataclass(frozen=True, slots=True)
class ProposalCard:
    """Mandatory metadata for any harness candidate promotion.

    All fields are required. Any dataclass instantiation missing a field
    raises TypeError, which is the desired enforcement at construction time.
    """

    target_surface: HarnessSurface
    evidence_record_ids: tuple[str, ...]
    expected_delta: float
    target_agent: str  # e.g. "eibrain", "openclaw", "mcp_consumer"
    risk_tier: str  # "L0" | "L1" | "L2" | "L3" | "L4"
    rollback_plan: str
    diff_lines: int
    diff_tokens: int
    notes: str = ""
    safety_wire: tuple[str, ...] = field(default_factory=tuple)  # required for L3+


class GateVerdict(str, Enum):
    ACCEPT = "ACCEPT"
    WARN = "WARN"
    REJECT = "REJECT"


@dataclass(frozen=True, slots=True)
class GateResult:
    verdict: GateVerdict
    reason: str
    held_in_score: float | None
    held_out_score: float | None
    delta: float | None
```

@dataclass(slots=True)
class HarnessGate:
    """Dual-regression gate: held-in ∩ held-out ∩ delta≥0.

    Acceptance rules:
        held_in delta < 0  -> REJECT
        held_out delta < 0 -> REJECT
        one split missing  -> WARN (configurable; default is fallback to old soft gate)
        both splits up     -> ACCEPT
    """

    card: ProposalCard
    allow_warn_on_missing: bool = True

    def evaluate(
        self,
        *,
        held_in_scores: dict[str, float],
        held_out_scores: dict[str, float] | None,
        baseline_held_in: float,
        baseline_held_out: float | None,
    ) -> GateResult:
        held_in_now = float(held_in_scores.get("accuracy") or 0.0)
        held_in_delta = held_in_now - float(baseline_held_in)

        if held_in_delta < 0:
            return GateResult(
                verdict=GateVerdict.REJECT,
                reason=f"held_in regressed: {held_in_delta:.4f}",
                held_in_score=held_in_now,
                held_out_score=None,
                delta=held_in_delta,
            )

        if held_out_scores is None or baseline_held_out is None:
            if self.allow_warn_on_missing:
                return GateResult(
                    verdict=GateVerdict.WARN,
                    reason="held_out split missing — falling back to held-in-only check",
                    held_in_score=held_in_now,
                    held_out_score=None,
                    delta=held_in_delta,
                )
            return GateResult(
                verdict=GateVerdict.REJECT,
                reason="held_out split required but missing",
                held_in_score=held_in_now,
                held_out_score=None,
                delta=held_in_delta,
            )

        held_out_now = float(held_out_scores.get("accuracy") or 0.0)
        held_out_delta = held_out_now - float(baseline_held_out)
        if held_out_delta < 0:
            return GateResult(
                verdict=GateVerdict.REJECT,
                reason=f"held_out regressed: {held_out_delta:.4f}",
                held_in_score=held_in_now,
                held_out_score=held_out_now,
                delta=min(held_in_delta, held_out_delta),
            )

        min_delta = min(held_in_delta, held_out_delta)
        return GateResult(
            verdict=GateVerdict.ACCEPT,
            reason=f"held_in={held_in_delta:.4f} held_out={held_out_delta:.4f}",
            held_in_score=held_in_now,
            held_out_score=held_out_now,
            delta=min_delta,
        )
```

def enforce_diff_size(*, diff_lines: int, diff_tokens: int) -> None:
    if int(diff_lines) > MAX_DIFF_LINES:
        raise ValueError(f"diff_lines {diff_lines} exceeds MAX_DIFF_LINES={MAX_DIFF_LINES}")
    if int(diff_tokens) > MAX_DIFF_TOKENS:
        raise ValueError(f"diff_tokens {diff_tokens} exceeds MAX_DIFF_TOKENS={MAX_DIFF_TOKENS}")


def enforce_diversity(candidates: list[dict[str, Any]], *, min_count: int) -> None:
    distinct_keys = {str(c.get("source_key") or c.get("id") or "") for c in candidates}
    if len(distinct_keys) < min_count:
        raise ValueError(
            f"need ≥{min_count} diverse candidates (source_keys), got {len(distinct_keys)}"
        )


def enforce_one_active_per_surface(
    *, new_surface: str, active_surfaces: list[dict[str, Any]]
) -> None:
    for entry in active_surfaces:
        if str(entry.get("target_surface") or "") == str(new_surface):
            raise ValueError(f"surface {new_surface!r} already has active candidate {entry.get('id')}")
```

    held_out_scores: dict[str, float] | None,
    baseline_held_in: float,
    baseline_held_out: float | None,
    scope: dict[str, Any] | None = None,
) -> dict[str, Any]:
    """Dual-regression evaluation. Returns ACCEPT / WARN / REJECT.

    Lazy-imports harness_patch to keep this module importable under 1.5.x
    default mode (when HARNESS_PATCH_V2 is unset).
    """
    from eimemory.governance.harness_patch import HarnessGate, HARNESS_PATCH_V2  # noqa: WPS433

    candidate = runtime.store.get_by_id(candidate_id, scope=scope)
    if candidate is None:
        raise ValueError(f"candidate not found: {candidate_id}")

    card_data = (candidate.content or {}).get("proposal_card") if HARNESS_PATCH_V2 else None
    if HARNESS_PATCH_V2 and not card_data:
        return {
            "verdict": "REJECT",
            "reason": "proposal_card missing on candidate",
            "candidate_id": candidate_id,
        }

    from eimemory.governance.harness_patch import HarnessSurface, ProposalCard  # noqa: WPS433

    if HARNESS_PATCH_V2:
        card = ProposalCard(
            target_surface=HarnessSurface(card_data["target_surface"]),
            evidence_record_ids=tuple(card_data.get("evidence_record_ids") or ()),
            expected_delta=float(card_data.get("expected_delta") or 0.0),
            target_agent=str(card_data.get("target_agent") or ""),
            risk_tier=str(card_data.get("risk_tier") or "L0"),
            rollback_plan=str(card_data.get("rollback_plan") or ""),
            diff_lines=int(card_data.get("diff_lines") or 0),
            diff_tokens=int(card_data.get("diff_tokens") or 0),
        )
    else:
        # Legacy mode: synthesize a minimal card for the gate
        from eimemory.governance.harness_patch import HarnessSurface as _HS  # noqa: WPS433
        card = ProposalCard(
            target_surface=_HS.RUNTIME_POLICY,
            evidence_record_ids=(candidate_id,),
            expected_delta=0.0,
            target_agent="legacy",
            risk_tier="L0",
            rollback_plan="legacy",
            diff_lines=0,
            diff_tokens=0,
        )

    gate = HarnessGate(card)
    result = gate.evaluate(
        held_in_scores=held_in_scores,
        held_out_scores=held_out_scores,
        baseline_held_in=baseline_held_in,
        baseline_held_out=baseline_held_out,
    )
    return {
        "verdict": result.verdict.value,
        "reason": result.reason,
        "held_in_score": result.held_in_score,
        "held_out_score": result.held_out_score,
        "delta": result.delta,
        "candidate_id": candidate_id,
    }
```

REQUIRED_SAFETY_MODULES_L3_PLUS = (

def _check_safety_wire(*, authority_tier: str, safety_wire: tuple[str, ...] | list[str]) -> None:
    tier = str(authority_tier or "").upper()
    if tier not in {"L3", "L4"}:

        raise ValueError(
            f"safety_wire missing required modules for {tier}: {sorted(missing)}"
        )
