# eimemory 0.10.1 Autonomous Learning Loop Implementation Plan

> **For agentic workers:** Use superpowers-open:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an autonomous learning kernel to eimemory 0.10.1 so the agent can maintain a self-model, generate its own learning goals, research independently, run offline experiments, evaluate candidates, and distill winning changes into reviewable capability assets.

**Architecture:** Build on the existing `EvolutionAPI`, `rule_evolution`, intake, evaluation, and scheduler layers. eimemory remains the memory/evolution substrate; it does not send external messages, spend money, mutate system prompts, or execute device/account actions. LLMs are injected tools for ranking, synthesis, and candidate generation, with deterministic fallbacks and explicit risk gates.

**Tech Stack:** Python 3.13, existing eimemory Runtime/RuntimeStore, JSONL/SQLite store abstractions, pytest, existing CLI and scheduler patterns.

---

## 10-Pass Review: What Was Missing

### Pass 1: Autonomy Boundary

Missing: previous plan still waited for task outcomes. 0.10.1 must let the system create its own learning agenda.

Decision: introduce `AutonomousLearningLoop`, not a daily retrospective loop.

### Pass 2: Self-Model

Missing: no durable map of what the agent can do, cannot do, and repeatedly does poorly.

Decision: add capability, weakness, and learning-debt records.

### Pass 3: Curiosity

Missing: no goal generator.

Decision: add a curiosity engine that proposes learning goals from self-model gaps, outcome traces, source discovery, user priorities, and recent world changes.

### Pass 4: World Observation

Missing: the agent needs proactive input channels beyond user-provided links.

Decision: use the existing intake/source discovery layer as the safe world observation surface; external fetching stays bounded and auditable.

### Pass 5: Research

Missing: autonomous learning requires evidence collection, not just LLM guesses.

Decision: add research artifacts with source URLs, summaries, applicability score, cost, risk, and replay idea.

### Pass 6: Offline Experimentation

Missing: no sandbox where new skill/SOP/tool/eval/prompt-routing candidates can be tried without changing production behavior.

Decision: add experiment records and candidate bundles; all experiments are file/store artifacts until promoted.

### Pass 7: Evaluation

Missing: no proof that a candidate makes the agent smarter.

Decision: extend evaluation with autonomous learning benchmarks: capability score, replay score, safety score, cost score, and regression score.

### Pass 8: Distillation

Missing: no bridge from research to durable capability.

Decision: add capability candidates that can become skill patches, SOP patches, tool-routing rules, eval cases, memory rules, or source policies.

### Pass 9: Safety

Missing: “self-evolution” can become unsafe if it mutates too much.

Decision: strict risk tiers:

- `L0`: write audit/report/candidate/eval artifacts only.
- `L1`: review-required changes to memory rules, SOP, skill drafts, tool routing, or source policy.
- `L2`: forbidden without explicit user approval: system prompt, external sends/posts, spend, account auth, private data export, device action, production code merge.

### Pass 10: Full-Version Path

Missing: 0.10.1 should be a real kernel, not a toy.

Decision: 0.10.1 implements the full loop in bounded local mode. Later versions add stronger world watchers, richer LLM optimizers, multi-agent research, and automatic PR generation.

```python
from __future__ import annotations

from typing import Any

from eimemory.models.records import ScopeRef


def build_self_model(runtime: Any, *, scope: dict, limit: int = 500) -> dict:
    scope_ref = ScopeRef.from_dict(scope)
    reflections = runtime.store.list_records(kinds=["reflection"], scope=scope_ref, limit=limit)
    rules = runtime.store.list_records(kinds=["rule"], scope=scope_ref, limit=limit)
    replays = runtime.store.list_records(kinds=["replay_result"], scope=scope_ref, limit=limit)

    weaknesses = []
    for record in reflections:
        tag = str(record.meta.get("tag") or record.content.get("tag") or "general")
        lesson = str(record.meta.get("fix") or record.content.get("fix") or record.summary)
        if lesson:
            weaknesses.append(
                {
                    "kind": tag,
                    "lesson": lesson,
                    "source_record_id": record.record_id,
                }
            )

    capabilities = [
        {
            "kind": str(rule.meta.get("task_type") or "general"),
            "title": rule.title,
            "status": rule.status,
            "source_record_id": rule.record_id,
        }
        for rule in rules
    ]

    replay_pass_rate = 0.0
    if replays:
        passed = sum(1 for item in replays if str(item.meta.get("verdict") or "") == "pass")
        replay_pass_rate = round(passed / len(replays), 3)

    return {
        "scope": dict(scope),
        "capabilities": capabilities,
        "weaknesses": weaknesses,
        "metrics": {
            "active_rule_count": sum(1 for item in rules if item.status == "active"),
            "reflection_count": len(reflections),
            "replay_count": len(replays),
            "replay_pass_rate": replay_pass_rate,
        },
    }
```

## 0.10.1 Acceptance Criteria

0.10.1 is complete only when all of these are true:

- [ ] `eimemory learn cycle` produces three learning goals without user input.
- [ ] The loop selects one goal and creates a research note.
- [ ] The loop creates one sandbox experiment.
- [ ] The loop evaluates the experiment offline.
- [ ] Passing experiments become `capability_candidate` records.
- [ ] No L1 candidate is applied without review.
- [ ] L2 actions are blocked by default.
- [ ] Governance snapshot exposes latest learning status.
- [ ] Nightly jobs can include the learning loop behind a config flag.
- [ ] Tests prove the loop can run without a real LLM.

## Full-Version Push

0.10.1 should implement the full architecture in local bounded form, not a fake pilot. The only things deferred are scale and autonomy level:

- Defer broad internet crawling.
- Defer automatic PR creation.
- Defer production prompt mutation.
- Defer external sends/spend/device/account actions.
- Do not defer self-model, curiosity, research records, experiments, eval, and distillation.

This means 0.10.1 is the first real autonomous learning version.
