> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make EIMemory 1.4.4 evidence-first, replay-proven, and capable of real autonomous code evolution with automatic patch, deploy, and rollback gates.

**Architecture:** Keep the productive loop small: memory retrieval, real task replay, autonomous patch, safety gate, deployment/rollback. Replace impression-based status answers with query-first evidence snapshots, turn known failures into replay cases, and make autonomous code patches prove themselves before and after production deployment.

**Tech Stack:** Python 3.11+, pytest, systemd user service on honxin, existing EIMemory runtime/store/governance modules.

---

## Scope

This version combines the operator's three requirements and the prior eight improvement points:

- Evidence-first: status/version/evaluation/deployment answers must query source of truth first.
- Real replay: prior mistakes become regression cases and must replay after fixes.
- Simplified evolution loop: keep memory retrieval, task replay, autonomous patch, safety gate; disable or demote modules with no online evidence.
- Patch generation: failure -> locate -> structured code patch -> verification -> commit/deploy.
- Failure sources: nightly, benchmark, OpenClaw, RPC health, systemd journal, user correction.
- Repo index: map files/functions/tests for patch targeting.
- Verification tiers: targeted tests, smoke replay, health checks.
- Rollback: commit/release/service rollback evidence.
- Ledger: record why the patch happened and whether it recurred.
- Production mode: honxin runs fully autonomous mode when explicitly configured.

```python
from __future__ import annotations

from dataclasses import dataclass
from typing import Any


QUERY_FIRST_KINDS = {"status", "version", "evaluation", "deployment"}


@dataclass(frozen=True)
class EvidenceQuery:
    kind: str
    source: str
    required: bool = True


def require_query_first(subject: str, queries: list[EvidenceQuery], *, evidence: dict[str, Any]) -> dict[str, Any]:
    facts: dict[str, Any] = {}
    missing: list[str] = []
    for query in queries:
        payload = evidence.get(query.source)
        if not payload:
            if query.required:
                missing.append(query.source)
            continue
        if isinstance(payload, dict):
            facts.update({key: value for key, value in payload.items() if value not in (None, "")})
    if missing:
        return {
            "ok": False,
            "subject": subject,
            "query_first": True,
            "blocked_reason": f"missing_required_evidence:{missing[0]}",
            "missing_sources": missing,
            "facts": {},
        }
    return {"ok": True, "subject": subject, "query_first": True, "blocked_reason": "", "facts": facts}
```

```python
from __future__ import annotations

from typing import Any


PRODUCTIVE_MODULES = ["memory_retrieval", "task_replay", "autonomous_patch", "safety_gate"]


def classify_evolution_modules(*, online_evidence: dict[str, dict[str, Any]]) -> dict[str, Any]:
    keep = [name for name in PRODUCTIVE_MODULES if int((online_evidence.get(name) or {}).get("success_count") or 0) > 0]
    for name in PRODUCTIVE_MODULES:
        if name not in keep:
            keep.append(name)
    demote = sorted(name for name, evidence in online_evidence.items() if name not in PRODUCTIVE_MODULES and int(evidence.get("success_count") or 0) <= 0)
    observe = sorted(name for name in online_evidence if name not in keep and name not in demote)
    return {"ok": True, "keep": keep, "demote": demote, "observe": observe}
```
